🔍 The Honest Truth About Web App Security

Your Firebase config (apiKey, projectId) cannot be fully hidden in any web app — not even apps built by Google. Anyone opening DevTools can see it. This is normal and not dangerous, because the API key alone is useless without passing Firestore Security Rules. Your real protection is the rules + App Check + API key restrictions working together. This guide sets up all of them.

Firebase Console
Firestore Security Rules
The single most important protection. Even with your API key, nobody can touch your data.
🔴 CRITICAL — Do This First
⚠️ Default rules are WIDE OPEN
When you first create Firestore, the default rules allow anyone in the world to read and write your database. You must replace them immediately.
1
Go to console.firebase.google.com and click your project.
2
In the left sidebar, click Build → Firestore Database.
3
Click the Rules tab at the top of the Firestore page.
4
Select ALL the existing text and delete it. Then paste the rules below exactly.
Firestore Rules
// CashFlow — Secure Firestore Rules
// Only YOU (signed in with YOUR Google account) can access your data.
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Your personal data — only you can read/write
    match /users/{userId}/{document=**} {
      allow read, write: if
        request.auth != null
        && request.auth.uid == userId
        && request.auth.token.email_verified == true;
    }

    // Block everything else — no exceptions
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
5
Click the Publish button (blue button top right). Done.
✅ What this does
request.auth.uid == userId means each user can only see their own data.
email_verified == true means only Google-verified accounts work — blocks throwaway accounts.
allow read, write: if false blocks everything at the root level as a fallback.

Google Cloud Console
Restrict Your API Key to Your Domain
Your API key will only work when requests come from your GitHub Pages URL — not from anyone else's computer.
🔴 CRITICAL
1
Go to console.cloud.google.com
2
At the top, click the project selector dropdown → select your Firebase project name.
3
Left sidebar → APIs & Services → Credentials
4
Under API Keys, you'll see a key called "Browser key (auto created by Firebase)". Click it.
5
Under "Application restrictions", select "HTTP referrers (websites)".
6
Under "Website restrictions", click Add an item and add these two entries:
https://YOUR_USERNAME.github.io/*
http://localhost/*
Replace YOUR_USERNAME with your actual GitHub username. The localhost entry lets you test locally.
7
Click Save. Changes take up to 5 minutes to activate.
✅ Result
If someone copies your API key and tries to use it in their own app or Postman, they get: API key not valid. Please pass a valid API key.

Firebase Console
Enable Firebase App Check
Makes Firebase verify that requests come from your actual app — not scripts, bots, or other apps using your key.
🟡 HIGH PRIORITY
1
In Firebase Console → left sidebar → Build → App Check
2
Click "Get started"
3
Click your web app (the </> icon one). Choose reCAPTCHA v3 as the provider.
4
Open a new tab → go to google.com/recaptcha/admin → sign in → click "+" to create new site.

Fill in:
• Label: CashFlow
• reCAPTCHA type: Score based (v3)
• Domains: add YOUR_USERNAME.github.io and localhost

Click Submit → copy the Site Key (starts with 6L...)
5
Back in Firebase App Check → paste the Site Key → click Save.
6
Click the "Enforce" button next to Firestore. This means only your app can write data.
7
In your CashFlow app: open Settings → scroll to bottom → tap "Set App Check Key". Or open DevTools → Console → type:
localStorage.setItem('cf_recaptcha_key', 'YOUR_SITE_KEY_HERE')
The app is already coded to use this key automatically once you store it.

Firebase Console
Lock Down Authorised Domains
Prevent sign-ins from any website except your own.
🟡 HIGH PRIORITY
1
Firebase Console → Build → Authentication → Settings tab
2
Under "Authorised domains", you'll see a list. Make sure it contains only these:
  • localhost
  • YOUR_USERNAME.github.io
Delete anything else you don't recognise.
3
Under "User account linking" → enable "Prevent creation of multiple accounts with same email".
✅ What this does
Even if someone clones your app and hosts it on their own website, Google Sign-In will refuse to work on their domain.

Google Cloud Console
Set Budget & Usage Alerts
Get an email the moment anything suspicious happens — before it costs you money.
🟡 HIGH PRIORITY
1
Go to console.cloud.google.com/billing
2
Click your billing account → left sidebar → Budgets & alerts → Create budget
3
Fill in:
  • Name: CashFlow Alert
  • Scope: select your Firebase project
  • Budget amount: $1 USD (you should NEVER reach this for personal use)
4
Under Alert thresholds, add: 50%, 90%, 100%. Make sure "Email alerts to billing admins" is checked.
5
Click Finish. You'll get an email if your Firebase usage ever spikes unexpectedly.
ℹ️ Free tier limits (very generous for personal use)
Firestore: 50,000 reads/day, 20,000 writes/day, 1 GB storage.
You personally will use maybe 200–500 reads/day. A spike to 10,000+ = suspicious.

GitHub Settings
Enable GitHub Security Scanning
GitHub will email you if you accidentally commit any passwords or secrets to your repo.
🔵 RECOMMENDED
1
Go to your cashflow repo on GitHub → click Settings tab
2
Left sidebar → Code security and analysis
3
Enable all of these:
  • ✅ Dependency graph
  • ✅ Dependabot alerts
  • ✅ Secret scanning — alerts you if any API key is found in your commits
4
Also make sure your repo is set to Public (required for GitHub Pages free tier) but confirm the only file in the repo is your app code — no .env files, no config files with real keys.
Your Firebase config is stored in the browser's localStorage, NOT in any file in your GitHub repo. That is correct and safe.
5
Upload the _headers file (included in your download) to your GitHub repo alongside index.html. This adds browser-level security headers to every page load.

Your Google Account
Secure Your Google Account (Master Key)
Your Google account IS your login. If it's compromised, everything is compromised. This takes 3 minutes.
🔴 CRITICAL
1
Go to myaccount.google.com/security
2
Under "How you sign in to Google" → click 2-Step Verification → turn it ON.
Use Google Authenticator app or a physical security key — not SMS if possible (SMS can be SIM-swapped).
2FA means even if someone gets your password, they can't sign in without your phone.
3
Under "Your devices" — review the list. If you see any device you don't recognise, click it → Sign out.
4
Under "Third-party apps with account access" — review the list. Remove anything you don't recognise.
5
Run Google's Security Checkup — it takes 2 minutes and tells you exactly what to fix.

Ongoing Monitoring
How to Know If You've Been Hacked
Check these 4 places. Takes 3 minutes once a month.
🔵 MONTHLY HABIT
👤 Firebase Auth Users

Firebase Console → Authentication → Users

You should only see yourself (1 Google account). Unknown emails = someone got in.

📊 Firestore Usage

Firebase Console → Firestore → Usage

Normal: ~200–500 reads/day. A spike to 5,000+ when you're not using the app = suspicious.

💸 Billing

console.cloud.google.com → Billing

You should always see $0.00. Any charge = abuse is happening.

🔐 In-App Security Log

CashFlow app → Settings → Security Centre

See every sign-in, edit, and suspicious event. Also run the Security Check button.

WHERE TO LOOK IN FIREBASE CONSOLE — STEP BY STEP
1
Check for unknown users: Firebase Console → Authentication → Users tab → count the rows. Should be exactly 1 (you).
2
Check read/write counts: Firebase Console → Firestore Database → Usage tab → look at the daily graph. Normal is a flat low line.
3
Check API usage: Google Cloud Console → APIs & Services → Dashboard → look at "Requests" numbers for Cloud Firestore API.
4
Check billing: Google Cloud Console → Billing → Overview. Should say $0.00 under "Cost to date this month".
5
Check your Google account: myaccount.google.com → Security → "Recent security activity". Look for sign-ins from unknown locations.

Emergency Response
If You Think You've Been Hacked — Do This NOW
Act fast. Follow these steps in order.
🚨 EMERGENCY PLAN
1
Sign out of CashFlow immediately
Open the app → Settings → Security Centre → tap "Emergency Sign Out". Or just close the app.
2
Change your Google account password
Go to myaccount.google.com → Security → Password → change it to something strong. This invalidates all existing sessions everywhere.
3
Delete any unknown Firebase users
Firebase Console → Authentication → Users → if you see any email that isn't you → click the 3-dot menu → Delete account.
4
Revoke and regenerate your API key
Google Cloud Console → APIs & Services → Credentials → click your Browser key → click "Regenerate key" → confirm. Then open your CashFlow app → paste new config in Setup screen.
5
Re-verify your Firestore rules
Firebase Console → Firestore → Rules → make sure the rules from Step 1 of this guide are still there and not modified.
6
Check what data was accessed
Firebase Console → Firestore → Usage tab → check the timeline. Firebase does not log individual document reads by default, but usage volume shows if data was extracted.
7
Enable 2FA if you haven't
myaccount.google.com → Security → 2-Step Verification → turn it ON. This is the single most effective thing you can do.
